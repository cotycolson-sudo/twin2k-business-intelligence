# twin2k-business-intelligence

Business intelligence analysis of the **Twin-2K-500** dataset — a synthetic panel of ~2,000 personas with rich demographic profiles and psychological/behavioral scores.

## Repository Layout

```
twin2k-business-intelligence/
├── data/
│   ├── raw/                   # Source parquet files (persona_chunk_001–007.parquet)
│   └── processed/
│       └── twin2k_flat.csv    # Flat CSV generated by extract_scores.py
├── deliverables/              # Final reports, exports, and dashboards
├── jmp/
│   └── notes.md              # JMP analysis setup and workflow notes
├── scripts/
│   └── extract_scores.py     # ETL script: parquet → flat CSV
└── tableau/
    └── notes.md              # Tableau dashboard setup and workflow notes
```

## Quick Start

### 1. Install dependencies

```bash
pip install pandas pyarrow
```

### 2. Add raw data

Place the seven source parquet files in `data/raw/`:

```
data/raw/persona_chunk_001.parquet
data/raw/persona_chunk_002.parquet
...
data/raw/persona_chunk_007.parquet
```

### 3. Generate the flat CSV

```bash
python scripts/extract_scores.py
```

Output: `data/processed/twin2k_flat.csv`

## Data

### Source files

Seven parquet chunks (`persona_chunk_001.parquet` through `persona_chunk_007.parquet`) each containing:
- `pid` — participant identifier
- `persona_summary` — free-text narrative containing demographic labels and score assignments

### Processed CSV (`twin2k_flat.csv`)

One row per participant. Columns:

| Group | Columns |
|---|---|
| Identifier | `pid` |
| Demographics | `geographic_region`, `gender`, `age`, `education_level`, `race`, `marital_status`, `religion`, `religious_attendance`, `political_affiliation`, `income`, `political_views`, `employment_status` |
| Big Five | `score_extraversion`, `score_agreeableness`, `wave1_score_conscientiousness`, `wave2_score_conscientiousness`, `score_openness`, `score_neuroticism` |
| Cognitive | `score_needforcognition`, `score_fluid`, `score_crystallized`, `crt2_score`, `score_syllogism_merged`, `score_wason`, `score_deductive_certainty`, `score_finliteracy`, `score_numeracy` |
| Social / Behavioral | `score_agency`, `score_communion`, `score_BES`, `score_socialdesirability`, `score_selfmonitor`, `score_SCC`, `score_GREEN`, `score_minimalism` |
| Decision-making | `score_actual_total`, `score_overconfidence`, `score_overplacement`, `score_ultimatum_sender`, `score_ultimatum_accepted`, `score_mentalaccounting`, `score_discount`, `score_presentbias`, `score_riskaversion`, `score_lossaversion`, `score_trustgame_sender`, `score_trustgame_receiver`, `score_dictator_sender` |
| Personality (other) | `score_anxiety`, `score_depression`, `score_HI`, `score_HC`, `score_VI`, `score_VC`, `score_RFS`, `score_ST-TW`, `score_CNFU-S`, `score_needforclosure`, `score_maximization`, `score_forwardflow` |

## Script Reference

### `scripts/extract_scores.py`

Parses `persona_summary` text from all seven parquet chunks and writes a single flat CSV.

**Configuration** (top of file):

| Variable | Default | Description |
|---|---|---|
| `DATA_RAW_PATH` | `data/raw` | Directory containing source parquet files |
| `DATA_PROCESSED_PATH` | `data/processed` | Output directory |
| `OUTPUT_FILENAME` | `twin2k_flat.csv` | Output file name |

**What it does:**
1. Loads and concatenates all seven parquet files
2. Validates that `pid` and `persona_summary` columns are present
3. Extracts demographic fields and numeric score variables from each `persona_summary`
4. Writes the flat CSV and prints a summary report with missing-value counts
